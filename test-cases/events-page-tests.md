## **TC-1: Перевірка відкриття меню фільтрації**

**Priority:** High  
**Pre-Conditions:**
1. Користувач не авторизований.
2. Сторінка **Events** відкрита.
3. Елементи фільтрації завантажені.

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | Натиснути на фільтр **"Тип події"** | — | Відкривається меню з варіантами вибору. |
| 2 | Перевірити наявність списку опцій | — | Усі доступні категорії відображаються у випадаючому списку. |
| 3 | Закрити сайт (завершити сесію) | — | Браузер закривається успішно. |

**Очікуваний результат:** Система коректно реагує на натискання кнопки. Меню відкривається швидко, без помилок у консолі.  
**Post-condition:** Сесія браузера завершена.

## **TC-2: Перевірка фільтрації подій за типом**
**Title (Summary):** Перевірка вибору конкретної категорії фільтрації "Економічний"  
**Priority:** High  
**Pre-Conditions:**
1. Користувач не авторизований.
2. Сторінка **Events** відкрита.
3. У системі існують події з категорією "Економічний".

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | Відкрити меню фільтра **"Тип події"** | — | Список категорій стає видимим для користувача. |
| 2 | Встановити позначку на чек-боксі **"Економічний"** | `Checkbox = Економічний` | Чек-бокс вибрано (стає активним); список подій оновлюється. |
| 3 | Закрити сайт (завершити сесію) | — | Браузер закривається успішно. |

**Очікуваний результат:** Система коректно позначає обраний фільтр. Користувачу відображаються події, що відповідають обраному критерію.  
**Post-condition:** Сесія браузера завершена.

## **TC-4: Пошук без результатів (Негативний)**
**Title:** Перевірка відображення сторінки при відсутності збігів у пошуку  
**Priority:** Medium  
**Pre-Conditions:**
1. Сторінка **Events** відкрита.
2. Пошукове поле порожнє.

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | Ввести у поле пошуку неіснуючий набір символів | `qwefyuvhl` | Текст введено; система ініціює пошук. |
| 2 | Перевірити повідомлення про відсутність результатів | — | Відображається текст: "Ми не знайшли жодних результатів...". |
| 3 | Перевірити стан лічильника подій | — | Лічильник оновлюється та показує: "Знайдено 0 подій". |

**Очікуваний результат:** Система коректно обробляє запит без збігів, надаючи користувачу зворотний зв'язок.
**Очікуваний результат:** Система коректно обробляє запит, на який немає збігів. Користувач отримує чітке повідомлення про відсутність результатів замість порожнього білого екрана.  
**Post-condition:** Натиснути на іконку "Х" у полі пошуку або видалити текст вручну для скидання результатів.
